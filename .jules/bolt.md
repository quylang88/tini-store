## 2024-03-20 - [Avoid Object.values().reduce() for calculating object property sums]
**Learning:** Using `Object.values(obj).reduce(...)` to calculate the sum of values in an object introduces overhead from intermediate array allocation (`Object.values()`) and callback execution (`reduce()`). Benchmarks show this is ~44x slower than a simple `for...in` loop with `Object.prototype.hasOwnProperty.call()` (207ms vs 4.68ms for 100k iterations).
**Action:** When calculating sums or aggregates over object properties in hot paths (like calculating total stock across warehouses which happens frequently during stock updates or data normalization), always use a `for...in` loop instead of `Object.values().reduce()`. I have extracted this pattern into a reusable `calculateTotalStock` helper in `src/utils/inventory/warehouseUtils.js`.
