## 2024-05-18 - [Date Comparison and Array Operations Optimization]
**Learning:** Reconstructing mapped array representations with `.filter()` followed by `.reduce()` chain on larger sets of objects allocates redundant intermediate arrays. The instantiation of `new Date()` multiple times inside these looping structures introduces high garbage collection load and CPU cost. `Date.parse()` is a lighter and quicker primitive for date validations inside loops.
**Action:** When filtering out arrays to compute properties inside a hook/component loop on high-frequency elements like `orders` or `products`, always consolidate nested operations using a single `for...of` sequence and primitive numbers logic (e.g., `Date.parse()` and `.getTime()`) replacing explicit `.filter()` `.reduce()` combinations along with objects instantiations.
