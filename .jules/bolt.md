## 2024-05-15 - Optimize reduce loops in utility functions
**Learning:** Found that `Object.values(obj).reduce(...)` and chained `.reduce()` on arrays is a common anti-pattern in the codebase that causes unnecessary intermediary memory allocations, which negatively impacts garbage collection and performance for hot-path utility functions.
**Action:** Default to using `for...in` combined with `Object.prototype.hasOwnProperty.call()` for summing/aggregating over large object values, and use `for...of` loops without callback allocation when processing arrays, directly mutating state when safe instead of functionally creating new objects.
