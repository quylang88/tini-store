## 2024-03-26 - [Optimize warehouse array operations to Maps and pre-allocated structures]
**Learning:** In a codebase heavily relying on constant dynamic array allocations (e.g. `Object.keys`, `Object.values().reduce`, or `.map`), converting to pre-calculated single-pass `Map` lookups, pre-allocated base objects (`EMPTY_STOCK`), and zero-allocation `for...in` loops drastically reduces Garbage Collection pressure and improves hot-path iteration speed. Codebase specific `getTotalStock` and `getWarehouseLabel` are heavily utilized, meaning avoiding `.find` overhead provides measurable speedups.
**Action:** When working on helper utilities that process frequently accessed data structures (like inventory state), prioritize single-pass IIFE initializations for caching maps over runtime array derivations. Always replace `Object.values(obj).reduce(...)` with a traditional `for...in` loop accompanied by `hasOwnProperty.call`.
