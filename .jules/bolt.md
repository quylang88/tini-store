## 2025-03-08 - Optimize React memoized values with large loops
**Learning:** In `src/screens/dashboard/StatsDetail.jsx`, the original `comparisonStats` calculation used chained array operations (`.filter().reduce()`) over large datasets, which resulted in significant overhead due to intermediate array allocations, multiple loop iterations, and object instantiations (`new Date(order.date)`).
**Action:** Replace `.filter().map().reduce()` chains within `useMemo` hooks with a single `for...of` loop when dealing with thousands of items to drastically reduce garbage collection pauses. Specifically, use `Date.parse(dateString)` instead of `new Date(dateString).getTime()` when checking date boundaries within a loop, as it sidesteps the relatively expensive object allocation and brings ~2x performance gains inside tight loops.
